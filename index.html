The Problem

Handling the increasing all the data and deliverables for a feature film has become a complex and difficult process, especially given the limited resource and rate at which the requirements change.. Part of the problem is the number of processes to manage, transfers to monitor, and reports to send out as each piece finishes. It's all automatable but not easy without creating a specific application to solve every workflow permutation.

That's where Flowpipe comes in. Flowpipe is an open-source application that enables you to build, change, and automate your media workflows with ease.

About Flowpipe

Flowpipe allows you to construct the pieces of your pipeline and automate the manual pieces of your workflows. The modular architecture makes it easy to add new steps, modify old workflows, or hook into a new web service as needed. As an example, here's a few workflows that could be automated:

  • Transcode Deliverables
  • Automatic Data Downloads / Uploads
  • VFX Pulls
  • Storing and Archiving Old Data

Plugins

Flowpipe's designed to be modular and easy to extend as needed. There's a number of third party services and tasks to create as pluggable pieces. To begin with, there's a few pieces that I'll prepare first:

  • EDL Parsing
  • ALE Parsing
  • Email
  • Reports
  • FTP / SFTP Protocols

Third party integrations will offer additional flexibility. Some longer term integration goals include:

  • FFMPEG
  • MediaSilo
  • PIX
  • REDCine / REDLine
  • ImageMagick

Who Am I?

Although I'll be doing this as a project separate from my full-time job, I am a Software Developer / Sys Admin for Light Iron. Light Iron is a production and post-production facility in Hollywood, CA. I started out in the Machine Room and have done all the manual tasks that I'm looking to automate with Flowpipe. I know from experience that building a tool that can handle diverse workflows and automate repeatable tasks will be invaluable in multiplying productivity, reducing errors, and answering the eternal question: "When will it be ready?"

I have worked on similar projects, such as Light Iron Sentinel. Sentinel is a portable webserver application that copies media, verifies its integrity, and guards media from acquisition to finishing. As with Flowpipe, the GUI is built entirely in HTML and CSS and it functions as a portable server. Sentinel has been a good learning experience and guide as to what works, what doesn't, and what matters.

Implementation

I'm planning to use in a new language called Go. Go runs quickly, supports parallel processing as a first-class citizen, works cross-platform, and can be distributed as a binary. As such, I think it will be an ideal candidate for this project. I'll be learning it as I go, but that's the way I have worked on all of my other projects so far. In addition, Go resembles Python and C. I've worked with Python for about 2 years now and feel comfortable the language and syntax. I also hope that, because it's open source, the results will be useful for others moving forward with Go.

FAQ

Why do you need to fund an open source project?
The goal of this is to guage interest and ensure that the problems we encounter are the same ones that others encounter. By supporting this project, you can signal that the problem is worth solving.
What does the license do?
The license enables the GUI for Mac, Windows, and Linux. The command line interface will be free and open-source and the GUI is designed to make it easier to use.
